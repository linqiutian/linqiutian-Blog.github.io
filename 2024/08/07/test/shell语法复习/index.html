Linux硬件开发常用Shell指令笔记

一、系统信息查看

  1. 查看系统信息

    • 解释：显示系统内核版本、主机名、硬件架构等详细信息。
    • 用途：确认系统环境是否符合开发要求。

  2. 查看CPU信息

    • 解释：显示CPU的详细信息，包括型号、核心数、主频等。
    • 用途：确认CPU性能是否满足需求。

  3. 查看内存信息

    • 解释：显示系统的内存信息，包括总内存、可用内存等。
    • 用途：评估内存是否足够支持程序运行。

  4. 查看系统负载

    • 解释：显示系统当前的负载情况，包括平均负载值。
    • 用途：监控系统运行状态，判断是否需要优化。

二、硬件设备检测

  1. 查看设备文件

    • 解释：列出系统中所有设备文件。
    • 用途：确认设备是否被正确识别。

  2. 查看USB设备

    • 解释：显示当前连接的USB设备信息。
    • 用途：检测USB接口的硬件设备是否连接成功。

  3. 查看PCI设备

    • 解释：显示系统中所有PCI设备信息。
    • 用途：确认PCI接口的硬件设备是否被识别。

  4. 查看硬件信息

    • 解释：详细列出系统中的硬件信息，包括CPU、内存、硬盘等。
    • 用途：全面了解硬件配置。

三、文件系统操作

  1. 查看文件系统

    • 解释：显示文件系统的磁盘使用情况，包括挂载点、总容量、已用容量等。
    • 用途：确认存储空间是否足够。

  2. 查看文件类型

    • 解释：显示文件的类型信息。
    • 用途：确认文件格式是否符合要求。

  3. 挂载文件系统

    • 解释：将设备挂载到指定的挂载点。
    • 用途：挂载外部存储设备。

  4. 卸载文件系统

    • 解释：卸载指定的挂载点。
    • 用途：安全地卸载外部存储设备。

四、网络操作

  1. 查看网络接口

    • 解释：显示网络接口的配置信息，包括IP地址、子网掩码等。
    • 用途：确认网络接口是否正常。注意：在现代Linux系统中，ip addr 更常用。

  2. 查看网络连接

    • 解释：显示当前的网络连接状态。
    • 用途：调试网络通信问题。注意：在现代Linux系统中，ss -an 更常用。

  3. 网络测试

    • 解释：测试与指定IP地址的网络连通性。
    • 用途：确认网络连接是否正常。

  4. 查看DNS配置

    • 解释：显示DNS服务器的配置信息。
    • 用途：确认DNS配置是否正确。

五、系统管理

  1. 查看进程

    • 解释：显示系统中所有进程的详细信息。
    • 用途：监控进程运行状态。

  2. 杀死进程

    • 解释：终止指定的进程。
    • 用途：停止异常运行的进程。

  3. 查看日志

    • 解释：查看系统日志文件。
    • 用途：调试系统运行中的问题。注意：在现代Linux系统中，journalctl 更常用。

  4. 更新系统

    • 解释：更新系统软件包。
    • 用途：确保系统环境是最新的。适用于基于Debian/Ubuntu的系统。

六、脚本与自动化

  1. 运行Shell脚本

    • 解释：执行Shell脚本文件。
    • 用途：自动化执行一系列命令。

  2. 查看脚本权限

    • 解释：查看脚本文件的权限。
    • 用途：确保脚本具有可执行权限。

  3. 设置脚本权限

    • 解释：为脚本文件设置可执行权限。
    • 用途：确保脚本可以正常运行。

七、硬件调试

  1. 查看串口设备

    • 解释：显示串口设备的初始化信息。
    • 用途：确认串口设备是否被正确识别。

  2. 查看GPIO状态

    • 解释：查看指定GPIO引脚的状态。
    • 用途：调试GPIO相关的硬件问题。

  3. 设置GPIO状态

    • 解释：设置指定GPIO引脚的状态。
    • 用途：控制GPIO引脚。

  4. 查看I2C设备

    • 解释：扫描I2C总线上的设备。
    • 用途：确认I2C设备是否被正确识别。

  5. 查看SPI设备

    • 解释：列出SPI设备文件。
    • 用途：确认SPI设备是否被正确识别。
